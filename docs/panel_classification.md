# Panel role classification — identity vs subclass vs activation state

## Why this exists
Whether a subpanel is "useful for phenotyping" is not a property of the panel's title — it depends on whether its genes **define what a cell is** (identity), **separate subtypes within a lineage** (subclass), or merely report an **inducible program a cell is running** (state). Reading the actual gene lists shows the XeniumPanelExplorer subpanels are functionally-themed bags that **mix all three**. This document defines a 3-tier framework, applies it gene-by-gene to the mixed panels, and records the resulting role for every subpanel.

Machine-readable companions (regenerated by `python3 scripts/build_panel_roles.py`):
- **`data/panel_roles.csv`** — one row per (tissue, subpanel): `tier`, `phenotyping_use`, `lineage_or_group`, `n_genes`, `n_identity_genes`, `phenotyping_source`, `note`.
- **`data/panel_gene_roles.csv`** — gene-level tier for **every gene of every cell-type identity panel** *and* the hand-curated mixed panels, with columns `gene_tier {identity|subclass|state|non_specific|epithelial_general}`, `lineage_hint`, `subtype_hint`, `cl_anchor`, `cl_terms`, `evidence {cellxgene|canonical|both}`, `note`.
- **`data/identity_core/<panel>.csv`** (shared) and **`data/tissues/<tissue>/identity_core/<panel>.csv`** — the `identity`-tier subset per panel: the genes a consumer scores (by **absolute** expression, not a panel mean) to call the lineage.
- **`data/identity_audit_<tissue>.csv`** — specificity check of each `identity_core` against the tissue's reference-run detection.
- **`data/identity_epithelial_general_<tissue>.csv`** — genes under `epithelial cell` but not a specific epithelial lineage (shared-epithelium embedding-extras).

The mixed/pathway panels keep the framework below; §"Gene-level identity classification" extends the same identity/subclass/state principle, gene-by-gene, to the cell-type *identity* panels using the Cell Ontology.

## Framework
According to PubMed, cell identity is best decomposed into **phenotype + lineage + state** (Morris 2019, *Development*, [doi:10.1242/dev.169748](https://doi.org/10.1242/dev.169748)); current best-practice annotation assigns **type** from stable identity markers and treats **state** as an orthogonal layer (Heumos et al. 2023, *Nat Rev Genet*, [doi:10.1038/s41576-023-00586-w](https://doi.org/10.1038/s41576-023-00586-w)).

| Tier | Definition | `phenotyping_use` |
|---|---|---|
| **Identity / lineage** | stable; defines what a cell *is* | `broad_mask` (per-cell argmax) or `embedding_extra` |
| **Subclass / fine-type** | separates subtypes *within* one lineage | `fine_tier` (only after the parent lineage is called) |
| **Activation state / program** | inducible, cross-lineage | `module_score` (score it; never a lineage axis). `exclude` if redundant/residual/custom |

**Operating rule:** assign lineage from Tier-1 masks; score Tier-3 programs separately (e.g. `AddModuleScore`); apply Tier-2 markers only conditioned on the parent lineage. This generalizes the instinct already in `Xenium_Analysis`, which excludes proliferation as "a cross-lineage PROGRAM, not a lineage" and tracks it via `CYCLING_MARKERS`.

## Redundancy findings and what was done
1. **Deleted** `data/tissues/thymus/subpanels/99_uncategorized.csv` — gene-identical to `99_uncategorized_REANNOTATED.csv` (verified: same 3,216 genes; the plain file only carried the derived `det_max`, recomputable from the two detection columns). Pancreas already omits the plain file; this aligns the two tissues.
2. **Residual nesting** 99 ⊃ 99c ⊃ 99d (perfect subsets in both tissues): kept REANNOTATED + 99c + 99d, all tagged `tier=residual, use=exclude`. They are audit states, not phenotyping panels.
3. **Pathway 21–49** tagged `module_score`; they overlap heavily *by design* (e.g. Reactome Immune System ⊃ Innate/Adaptive/Cytokine/Interleukin). Key them by **name, not number**, across tissues (pancreas 21 = glycolysis ≠ thymus 21 = HALLMARK mTORC1).
4. **Fat-vs-shared duplication**: phenotyping consumes only the `gene` column. `panel_roles.csv` records `phenotyping_source`, showing the pancreas generic lineages are sourced from `subpanels_shared/`; the per-tissue "fat" copies are app-only (in-app browser/compare) and carry detection columns the phenotyper never reads.
5. **Custom panel** left as-is (`exclude` for phenotyping). Asymmetry noted: thymus ships `50_custom_100_addon.csv` as a fat subpanel + `custom_panel.csv` at root; pancreas ships only `custom_panel.csv`.

## Gene-level identity classification for the cell-type panels (CL-based)

The verdicts above tier the *mixed* panels. The cell-type **identity** panels
(pancreas 01-07/11-13/15-16, thymus 01-09/12-16) were themselves only ever
*panel*-tiered — every gene in them was implicitly "identity". But these panels
are over-inclusive **feature pools**: padded with subclass markers, activation
genes, cell-cycle genes and broadly-expressed genes. Scoring a panel **mean**
therefore mislabels off-lineage cells — e.g. the 228-gene `04_immune_B_plasma`
panel carries cell-cycle genes (MKI67, TOP2A, CCNA2, CDK1, AURKB, BIRC5), so a
*proliferating acinar* cell scores high on the "B-cell" panel. The fix is to
answer, per gene, *"is this useful for **identity**?"* and emit a clean
`identity_core` the consumer scores by **absolute** expression.

**Method** (`scripts/derive_identity_panels.py`; hierarchy-aware, not string
matching). The CellxGene `cell_type` terms are Cell Ontology (CL) terms forming
an is-a graph (vendored release in `data/ontology/`, see its README). For each
gene we map its terms to CL ids and test descendancy: a term *supports* a
lineage iff it is a descendant of (or equal to) one of that lineage's CL
**anchors**. Subtypes roll up correctly (`pancreatic A cell` ⊂ `pancreatic
endocrine cell` ⊂ `endocrine cell`; both pancreas and thymus are **epithelial**
organs, so epithelial/keratinocyte terms are kept, not discarded).

| Lineage (panels) | CL anchor(s) | Subtype anchors → `subtype_hint` |
|---|---|---|
| Endocrine (panc 01) | pancreatic endocrine cell | pancreatic A/B/D/PP/epsilon cell → alpha/beta/delta/PP/epsilon |
| Exocrine (panc 02) | pancreatic acinar cell, pancreatic ductal cell | acinar / ductal |
| T (03), Thymocyte (thy 06-09) | T cell (+ thymocyte) | CD4/CD8/regulatory T; DN/DP thymocyte |
| B (04, thy 13) | B cell, plasma cell | plasma / memory / naive |
| NK/ILC (05, thy 14) | natural killer cell, innate lymphoid cell | ILC2 |
| Myeloid (06, thy 12) | myeloid leukocyte, dendritic cell, mast cell | macrophage/monocyte/cDC/pDC/mast |
| Granulocyte (07) | granulocyte | neutrophil/eosinophil/basophil |
| Vascular (11, thy 16) | endothelial cell | lymphatic |
| Mural (12) | pericyte, smooth muscle cell | pericyte / SMC |
| Fibroblast (panc 13, thy 15) | fibroblast, mesenchymal cell | stellate / myofibroblast |
| Neural — **peripheral** (15) | Schwann cell, peripheral nervous system neuron (NOT generic CNS `neuron`) | glia / neuron |
| Epithelial-general (16) | epithelial cell | — (embedding-extra, not a mask) |
| TEC (thy 01-04) | epithelial cell of thymus | cortical/medullary TEC; tuft/neuroendocrine/ionocyte |
| Hassall (thy 03) | keratinocyte, epithelial cell of thymus | keratinocyte |
| Myoid (thy 05) | muscle cell | skeletal |

**Tiers.** `identity` (terms dominantly under the anchor, ≥70% share);
`subclass` (supporting terms concentrated on one subtype anchor — carries
`subtype_hint`); `state` (a program gene — `cellchat_pathway`, a pathway-panel
member, or cell-cycle — that does not mark the lineage); `epithelial_general`
(under `epithelial cell` but not the panel's specific epithelial lineage —
shared epithelium); `non_specific` (maps to many lineages / broad terms, or
unannotated and not canonical).

**Canonical override (required).** ≈43% of the panel — including textbook
markers (IAPP, MAFA, NKX6-1, SLC30A8, KRT19, CHGA, TRAC) — has *no* `cell_type`.
A curated, literature-backed canonical list per lineage forces `identity`
regardless of CL and resolves multi-lineage mis-maps (MAFB = alpha-cell TF
though CL also lists myeloid; PMP22 = Schwann though listed broadly). `evidence`
records `cellxgene` / `canonical` / `both` (or `none` for an unannotated,
non-canonical gene).

**Cell-cycle override.** Proliferation is the textbook cross-lineage
contaminant, and CellxGene sometimes annotates a cell-cycle gene to a single
proliferating subtype (CCNB2, CEP55 → "DN4 thymocyte"), which would otherwise
read as identity. Membership in a cell-cycle panel or a curated proliferation
marker set forces `state`, overriding CL — but never a canonical identity
marker.

**Specificity validation (data-backed).** `identity_audit_<tissue>.csv` joins
each `identity_core` gene to its reference-run `detection_pct` and flags genes
detected in ≥50% of all cells as `ubiquitous_candidate_non_specific`. This only
*reports* candidates (the runs give section-level, not per-lineage, detection);
it does not auto-demote.

**Consumer use.** Score each panel's `identity_core` by **absolute** expression
to call the lineage cleanly (no catch-all), then refine with `subclass` markers;
score `state` panels separately (`AddModuleScore`); the full panels remain
available for embedding / feature learning. Residual caveat: a few genes
CellxGene mono-annotated to a lineage subtype (general/DNA-repair genes) can
still enter a core, but absolute-expression scoring of the canonical backbone
(CD3E/CD3G/LCK… for T, KIR/KLR/NCR1 for NK, …) dominates, so they do not cause
the catch-all mislabeling that the panel mean did.
